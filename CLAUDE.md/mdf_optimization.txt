# Project Context: Section Optimization for Structural Design

## 1. Project Purpose

This project is not just a Python coding exercise.

The purpose is to develop a structurally explainable section optimization process.  
The goal is to generate multiple section candidates, evaluate their structural performance, filter them by design constraints, and select the most efficient section based on material usage, weight, cost, or performance efficiency.

The final result should be explainable in structural engineering terms, not just as a numerical minimum.

## 2. What I Want to Build

I want to build a process that can answer the following questions:

1. What section candidates exist?
2. How were those candidates generated?
3. What structural properties are evaluated?
4. Which constraints must each section satisfy?
5. Why are some candidates rejected?
6. Why is the final selected section reasonable?
7. How can this process later connect to Midas or seismic design optimization?

## 4. Optimization Logic

The basic optimization process should be:

1. Generate section candidates.
2. Calculate or read structural properties for each candidate.
3. Apply strength, stiffness, displacement, and constructability constraints.
4. Filter out candidates that do not satisfy the constraints.
5. Define an objective function such as minimum material usage, minimum weight, minimum cost, or maximum efficiency.
6. Select the best candidate.
7. Save and explain the result.

A report-style explanation should be possible:

“Generated section candidates were evaluated based on structural performance indicators. Candidates satisfying strength and stiffness constraints were selected, and the optimal section was determined by minimizing material usage while satisfying all required design conditions.”
